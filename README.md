## Задание 1 — Шаблон Cache (Дробь)
<details>
<summary>Подробнее об задании</summary>

### Описание

Разработан класс **Fraction**, представляющий математическую дробь.

Главная цель задания — продемонстрировать использование шаблона проектирования **Cache**, позволяющего сохранять результат вычисления и повторно использовать его при последующих вызовах.

### Логика решения

Создан интерфейс **FractionValueable**, который содержит методы:

- получение вещественного значения дроби
- изменение числителя
- изменение знаменателя

Реализован класс **Fraction**, который хранит:

- числитель
- знаменатель

Класс реализует преобразование дроби в вещественное число.

Также создан класс **CachedFraction**, который наследуется от `Fraction` и реализует механизм кэширования результата метода `toDouble()`.

При первом вызове метода значение вычисляется и сохраняется.  
При повторных вызовах возвращается сохранённый результат.

Если числитель или знаменатель изменяются — кэш сбрасывается.

### Пример

```java
Fraction f = new Fraction(3, 4);
System.out.println(f.toDouble());

Кэшируемая дробь:

CachedFraction cf = new CachedFraction(7, 8);
System.out.println(cf.toDouble());
```

## Задание 2 — Структурный шаблон (Кот)

<details>
<summary>Подробнее об задании</summary>

### Описание

Реализована система подсчёта количества мяуканий кота без изменения исходного класса `Cat`.

Главная цель задания — продемонстрировать использование **структурного шаблона проектирования**, позволяющего добавить новое поведение объекту без изменения его исходного кода.

### Логика решения

Создан интерфейс **Meowable**, который содержит метод:   
void meow()

Класс **Cat** реализует обычное поведение кота и содержит метод `meow()`, выводящий сообщение о мяукании.

Для подсчёта количества мяуканий создан класс **MeowCounter**, который:

- принимает объект `Cat`
- реализует интерфейс `Meowable`
- перехватывает вызовы метода `meow()`
- увеличивает счётчик при каждом вызове

Таким образом объект кота **оборачивается** в дополнительный класс, который расширяет его функциональность.

Также создан сервисный класс **MeowService**, который принимает набор объектов типа `Meowable` и вызывает у них метод `meow()`.

### Пример

```java
Cat cat = new Cat("Барсик");
MeowCounter counter = new MeowCounter(cat);

MeowService.meowAll(counter, counter, counter);

System.out.println(counter.getMeowCount());

Результат выполнения покажет, сколько раз кот мяукнул.
```

</details> 

## Задание 3 — Коллекция List

<details>
<summary>Подробнее об задании</summary>

### Описание

Необходимо сформировать список **L**, содержащий элементы из списка **L1**, которые **не входят в список L2**.

Каждый элемент должен быть включён в результирующий список **только один раз**.

### Логика решения

Для выполнения задания создан класс **ListTask**.

В классе реализован метод:

```java
buildList(List<Integer> l1, List<Integer> l2)

Алгоритм работы метода:

Последовательно просматриваются все элементы списка L1

Для каждого элемента проверяется, содержится ли он в списке L2

Если элемент отсутствует в L2, он добавляется в результат

Для исключения повторов используется множество, поэтому каждый элемент включается только один раз

Таким образом формируется список, содержащий только уникальные элементы из L1, которых нет в L2.

Пример
L1 = [5, 1, 5, 2, 3, 2, 4]
L2 = [2, 4]

L = [5, 1, 3]
```
</details> 

## Задание 4 — Коллекция Map

<details>
<summary>Подробнее об задании</summary>

### Описание

Необходимо определить **номера школ**, в которых **средний балл выше среднего по району**.

Информация об учениках хранится в **текстовом файле**.  
Каждая запись содержит:

- фамилию
- имя
- номер школы
- балл

### Логика решения

Создан класс **StudentResult**, который хранит данные об ученике:

- фамилия
- имя
- номер школы
- балл

Создан класс **SchoolMapTask**, который выполняет обработку данных.

Алгоритм работы программы:

1. Чтение данных учеников из текстового файла
2. Вычисление **среднего балла по району**
3. Группировка учеников по номеру школы
4. Вычисление **среднего балла каждой школы**
5. Сравнение среднего балла школы со средним баллом по району
6. Формирование списка школ, у которых средний балл выше районного

Для хранения данных используется структура:

```java
Map<Integer, Double>

где:

ключ — номер школы

значение — средний балл школы

Пример результата
Средний балл по району: 85.3
Школы выше среднего: [18, 23]
```
</details> 

## Задание 5 — Коллекция Set

<details>
<summary>Подробнее об задании</summary>

### Описание

Файл содержит текст на русском языке.

Необходимо определить **символы**, которые встречаются **только в одном слове текста**.

### Логика решения

Для выполнения задания создан класс **UniqueWordSymbols**.

Алгоритм работы программы:

1. Текст считывается из файла
2. Полученная строка разбивается на отдельные слова
3. Для каждого слова формируется множество уникальных символов
4. С помощью структуры `Map<Character, Integer>` подсчитывается, в скольких словах встретился каждый символ
5. В итоговый результат попадают только те символы, которые встретились ровно в одном слове

Для хранения результата используется структура:

```java
TreeSet<Character>

Это позволяет автоматически получить отсортированный набор символов.

Пример

Текст:

мама мыла раму

Результат:

[л, р, у, ы]
```
</details>

## Задание 6 — Коллекция Queue

<details>
<summary>Подробнее об задании</summary>

### Описание

Необходимо переписать элементы из очереди **L1** в очередь **L2** в **обратном порядке**.

### Логика решения

Для выполнения задания создан класс **QueueReverser**.

Алгоритм работы программы:

1. Все элементы из очереди `L1` последовательно извлекаются
2. Извлечённые элементы помещаются во временный стек
3. Затем элементы из стека добавляются в очередь `L2`
4. За счёт принципа работы стека порядок элементов становится обратным

Таким образом получается новая очередь, содержащая элементы исходной очереди в перевёрнутом порядке.

### Пример

```text
L1 = [1, 2, 3, 4, 5]
L2 = [5, 4, 3, 2, 1]
```
</details>

### 7.1 Построение ломаной линии

Дан список объектов **Point**.

Необходимо:

- удалить повторяющиеся точки

- преобразовать отрицательные значения **Y** в положительные
- собрать результат в объект **Polyline**

Для обработки используется **Stream API**:

```java
distinct()
sorted()
map()
collect()
```
В результате формируется ломаная линия, содержащая обработанный список точек.

сгруппировать имена по номеру

Map<Integer, List<String>>


## Использование коллекций и Stream API

### Описание

В данной лабораторной работе были использованы возможности языка **Java** для работы с основными структурами данных из пакета **java.util**, а также механизм **Stream API** для обработки коллекций.

Коллекции позволяют эффективно хранить и обрабатывать группы объектов, а Stream API обеспечивает удобный и декларативный способ выполнения операций над наборами данных.

---

### Использованные структуры данных

В ходе выполнения лабораторной работы были применены следующие структуры данных:

**List**

Используется для хранения упорядоченных коллекций элементов.  
В задании 3 список применяется для формирования новой коллекции элементов, которые присутствуют в одном списке и отсутствуют в другом.

**Map**

Используется для хранения пар *ключ-значение*.  
В задании 4 структура `Map<Integer, Double>` применяется для хранения среднего балла каждой школы, где ключом является номер школы, а значением — вычисленный средний балл.

**Set**

Используется для хранения уникальных элементов.  
В задании 5 множество применяется для хранения уникальных символов слов, а также для формирования итогового набора символов без повторений.

**Queue**

Используется для организации структуры данных типа *очередь*.  
В задании 6 реализована операция разворота очереди с использованием дополнительной структуры данных (стека).

---

### Использование Stream API

В задании 7 были применены возможности **Stream API** для обработки коллекций.

Использованы следующие операции:

- `distinct()` — удаление повторяющихся элементов
- `sorted()` — сортировка элементов
- `map()` — преобразование элементов потока
- `collect()` — сбор результатов обработки

Stream API позволяет выполнять операции над коллекциями более компактно и декларативно.
